# Study Log Web App(v.2)

学習行動と日常の予定を統合的に記録し、
「どのような条件で学習の満足度（＝やる気）が高まるのか」を分析・可視化し、それに応じた学習(タスク)を提案するWebアプリです。

Version 1(元々作成しようとしていたもので、Notionに詳しく載せています)ではログの記録に重点を置いていましたが、Version 2では
タスク・予定・学習ログを統合し、行動改善までつなげる設計に進化させています。
## 開発方針
本プロジェクトでは、AIと対話しながら実装を進める「バイブコーディング」を採用しています。

この手法を採用した目的:  
実装を通して必要な技術を把握する  
試行錯誤しながら設計力を高める  

Version 2では、これに加えて以下を重視しています：  
UI/UXの一貫性    
将来的な機械学習への拡張性  

本プロジェクトの開発プロセスや、学習した技術（Flask, SQLite等）の詳細は、以下のNotionページで公開しています。  
[学習記録、試行錯誤はこちらから](https://www.notion.so/2c45d5e4275c8066a22cf18d7b4a2553?source=copy_link)

## 概要

-Flaskを用いた学習支援Webアプリ  
-学習ログ・タスク・予定を一元管理  
-SQLiteにデータを保存  
-学習満足度を基に行動改善を行う仕組みを構築する予定  

## 機能

① 学習ログ管理  
- 学習開始時刻    
- 学習時間  
- 満足度（1〜5）の記録（教師データとして利用)   
- メモの記録    
- ログの一覧表示
    
② タスク・予定管理  
- やること（Task)   
- 予定（Schedule）   
- 日付・締切の設定  
- メモの記録
  
③ データ可視化  
- 学習ログの一覧表示  
- 現在のやる気パラメータの表示（将来実装）  
- タスク・予定のカレンダー表示(将来実装)
  
④ やる気推定・行動提案（開発中）
- 学習ログを教師データとして使用   
- タスク・予定を特徴量として活用  
- 満足度（achievement）を予測  
- 状態に応じた最適なタスクを提案  

## 技術スタック

- Python  
- Flask  
- SQLite  
- HTML/CSS  

※ Version 2では、まずローカル環境での安定動作を優先し、Dockerは一時的に使用していません。
